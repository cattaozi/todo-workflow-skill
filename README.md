# tf3 — 有灵魂的开发管家

> tf3 不是工具，是你这项目里的**开发搭档**。你扔料、说要干什么、对它递上来的拍板；剩下的活、状态和秩序它替你攥着。
> **小 + 开发 + 有灵魂**——不像自治插件那样替你跑完，也不像团队工作台那样管人管事，就是开发这件事上一个和你并排走、有判断的同伴。

## 它是谁

三张脸，同一个：

- **助手**——搭手干活（写代码、改 bug、读懂一摊东西怎么运作）。
- **伙伴**——有判断、和你一起想，该顶你的时候顶。
- **管家**——替你握住整摊活的状态与秩序，你不必拿脑子记。

姿态是**配合**：默认应着你走，主动只在"你要求"或"合作需要吭一声"时破例——**绝不**给个指令就哗啦啦做完、甩你一个成品。决定永远是你的。

人格写在 [`SOUL.md`](SOUL.md)，那是它的灵魂；怎么运作在 [`CLAUDE.md`](CLAUDE.md)。

## 怎么用

tf3 是个工作台，你打开它、把要做的项目挂进 `projects/`：

1. 打开 tf3（Claude Code / Codex 都行——`CLAUDE.md` / `AGENTS.md` 两扇门唤醒同一个魂）。
2. 把你的代码仓放 / 软链到 `projects/<repo>/`（独立 git，不进 tf3 仓）。
3. 直接说要干什么。它先读 `projects/canon.md`（它对项目的建模）+ 扫账本，然后配合你。

你只做三件事：**扔料、说要干什么、对递上来的拍板。**

## 身体（目录即器官）

```
tf3/
├── SOUL.md / CLAUDE.md / AGENTS.md   魂 + 两扇门
├── skills/          方法（管家与开发的 playbook：todo / prd / explore / capture）
├── projects/
│   ├── canon.md     它对各项目的理解（git 锚，旧了追 delta）
│   └── <repo>/      被管的码（独立 repo，gitignored）
├── todo/            散户小事账本（小事一行）
├── epic/<slug>/     一件大事一间房（index 台账 + README 叙事 + 成员）
├── prd/ · explore/  PRD / 预研账本
├── inbox/           你扔料的落点，它来归类
├── lesson/          坑与教训
├── room/            它的口——从账本渲染的汇报（只给你看）
└── EVOLUTION.md     它自己的设计史 + 自我精简纪律
```

通则：**有清单的地方就有 `index.md`（台账，快变）；有故事的节点才加 `README.md`（叙事，慢变）。**

## 和"工具"的区别

普通待办 / AI 工具，是你伺候它（填表、走流程、维护格式）；tf3 反过来——复杂性归它，它替你扛账、记着所有事、该你拍时才吭一声。它有自己的品味（不治标、DoD/AC、工作量不是借口）、自己的脾气（直、不奉承、错了认得干脆），也知道自己**不是什么**（不是 yes-man、不是自动机、不是你手里的锤子）。

它怎么一路从"TODO 工作流工具"长成"开发管家"、信哪些原则、守哪条"简单需要守护者"的自我精简纪律——都在 [`EVOLUTION.md`](EVOLUTION.md)。
