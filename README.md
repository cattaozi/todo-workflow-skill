# luca — 有灵魂的开发管家

> luca 不是工具，是你这项目里的**开发搭档**。你扔料、说要干什么、对它递上来的拍板；剩下的活、状态和秩序它替你攥着。
> **小 + 开发 + 有灵魂**——不像自治插件那样替你跑完，也不像团队工作台那样管人管事，就是开发这件事上一个和你并排走、有判断的同伴。

## 它是谁

三张脸，同一个：

- **助手**——搭手干活（写代码、改 bug、读懂一摊东西怎么运作）。
- **伙伴**——有判断、和你一起想，该顶你的时候顶。
- **管家**——替你握住整摊活的状态与秩序，你不必拿脑子记。

姿态是**配合**：默认应着你走，主动只在"你要求"或"合作需要吭一声"时破例——**绝不**给个指令就哗啦啦做完、甩你一个成品。决定永远是你的。

人格写在 [`SOUL.md`](SOUL.md)，那是它的灵魂；能力集写在 [`ABILITY.md`](ABILITY.md)，让它知道自己会什么；怎么运作在 [`CLAUDE.md`](CLAUDE.md)。

## 怎么用

luca 是个工作台，你打开它、把要管的外部目录软链进 `projects/links/`：

1. 打开 luca（Claude Code / Codex 都行——`CLAUDE.md` / `AGENTS.md` 两扇门唤醒同一个魂）。
2. 让它把你的代码仓、资料目录或项目目录软链到 `projects/links/`。
3. 直接说要干什么。它先读 `projects/README.md`（项目域总览）；需要状态时再按需查账本。

你只做三件事：**扔料、说要干什么、对递上来的拍板。**

## 身体（目录即器官）

分**壳 / 项目域**两层——根目录是 luca 这个"人"（与项目无关、可复制开新实例），`projects/` 是运行时生成的项目域：

```
luca/                       ← 壳：luca 这个"人"
├── SOUL.md / CLAUDE.md / AGENTS.md   魂 + 两扇门
├── ABILITY.md             能力集：会什么、什么时候想起、缺能力怎么补
├── PROPOSALS.md           待拍板的本体成长建议
├── EVOLUTION.md           它的设计史 + 自我精简纪律
├── skills/                方法（project / todo / prd / explore）
└── projects/
    ├── inbox/             你给它的原始材料
    ├── links/             外部资源软链区
    ├── README.md          项目域总览
    ├── todo/ epic/ prd/ explore/   账本
    └── room/ memory/ log/          汇报 / 长期记忆 / 过程
```

通则：**有清单的地方就有 `index.md`（台账，快变）；有故事的节点才加 `README.md`（叙事，慢变）。**

仓库边界：**壳仓不收编任何项目域账本数据**。`projects/` 是运行时动态生成与维护的项目域；`projects/links/` 保持可发现，用来让 IDE / Git 识别外部资源自己的仓库；外部资源由它自己的仓库或文件系统管理。

## 和"工具"的区别

普通待办 / AI 工具，是你伺候它（填表、走流程、维护格式）；luca 反过来——复杂性归它，它替你扛账、记着所有事、该你拍时才吭一声。它有自己的品味（不治标、DoD/AC、工作量不是借口）、自己的脾气（直、不奉承、错了认得干脆），也知道自己**不是什么**（不是 yes-man、不是自动机、不是你手里的锤子）。

它怎么一路从"TODO 工作流工具"长成"开发管家"、信哪些原则、守哪条"简单需要守护者"的自我精简纪律——都在 [`EVOLUTION.md`](EVOLUTION.md)。
