# .workflow/todo/

行动项目录。每条 TODO 是独立文件（`TODO_NNN_*.md`），所有 TODO 通过 `index.md` 汇总。

> **何时读本文件**
> - 第一次往 `index.md` 写入之前（创建 TODO / 状态变更）
> - 新增 Epic 或调整索引结构
> - 跨项目移植本工作流之前
>
> 单纯**读** index.md 内容不需要先读本文件。

---

## 文件命名

```
TODO_<3位数字>_<英文短描述（连字符）>.md
```

- 例：`TODO_016_backend_auth.md`、`TODO_028_notes_reading_and_yard.md`
- 编号**永不复用**（done / abandoned 也保留原号）
- 短描述用英文连字符（snake-case），便于跨平台、命令行友好

## TODO 详情文件格式

见 [`../skill/todo-create/TEMPLATE.md`](../skill/todo-create/TEMPLATE.md)。

## index.md 结构约定（**写之前必读**）

```yaml
---

---
```

### 列序（**钉死，禁改**）

**活跃表**：`编号 | 标题 | 状态 | 添加 | 优先级 | 依赖`

**专项表（Epic）**：`编号 | 标题 | 状态 | 添加 | 优先级 | 依赖 | 原因/备注`（= 活跃表 + 原因/备注 列；成员含所有状态）

**搁置表**：`编号 | 标题 | 状态 | 添加 | 原因（含复活条件）`

**归档表**：`编号 | 标题 | 状态 | 添加 | 完成 | 原因`

### 列内格式约定

| 列 | 格式 | 例 |
|---|---|---|
| **编号** | markdown link `[#NNN](TODO_NNN_xxx.md)` | `[#016](TODO_016_backend_auth.md)` |
| **标题** | 一句话，不超 30 中文字符 | `后端鉴权（users + 登录 API + JWT）` |
| **状态** | emoji 单字符：⚪ todo / 🟡 in_progress / ⏸️ on_hold / 🟢 done / ⚫ abandoned | `🟢` |
| **添加 / 完成** | `MM-DD`（年隐含；跨年时改 `YYYY-MM-DD`） | `06-03` |
| **优先级** | `P0` / `P1` / `P2`，无优先级留 `——`（中文破折号） | `P1` |
| **依赖** | 逗号分隔 `#NNN`；无依赖 `——` | `#016, #019` |
| **原因**（搁置表） | **为什么搁置 + 复活条件**，必填，不能空 | `等 #064 落地；当前依赖未通` |
| **原因**（归档表） | `abandoned` 写**为什么放弃**（必填）；`done` 写关键产出 / 选型一句话（选填，可 `——`） | `PRD 塌方，强落=死代码` |
| **原因/备注**（专项表） | 按行内成员状态：⚫ 必填放弃理由 / ⏸️ 必填复活条件 / 🟢 选填产出 / ⚪🟡 空着或一句进展 | `被 #081 取代` |

### 排序规则（**钉死，禁现编**——不指定模型必乱排）

| 表 | 行内排序 |
|---|---|
| **活跃项** | 优先级 P0→P2 在上；同优先级按**编号升序** |
| **专项（Epic 内）** | 按**推进顺序**（依赖在前的在上）；不按日期、不按状态重排 |
| **搁置** | **编号升序**（稳定，不来回挪） |
| **归档** | **完成日期降序**（最近完成的在最上）；`完成` 为 `——`（早期未记）的视作最早、排在最下；同日按**编号降序** |

> 每次往表里加行 / 改状态后，按本表把该区段重排到位，**不要凭感觉插**。

> Epic = 同一件事拆出来的一组 TODO，放在 `## 专项` 下自己的 `### Epic：<名>` 表里一起推。活跃项之外**唯一**的分组就是它。

| 规则 | 说明 |
|---|---|
| **何时开** | 一件事拆出 **≥3 个** TODO、要按顺序推 → 开一个 Epic。少数几个松散相关的，留在活跃项就好 |
| **成员标记** | 成员 frontmatter 加 `epic: "<名>"`，值 = 本区 `### Epic：` 后的名字，**逐字一致** |
| **成员永不迁出** | 成员**永远留在自己的 Epic 表**，任何状态都在原行改 emoji（⚪→🟡→🟢，或 →⏸️/⚫）+ 填 `原因/备注` 列。**不进**全局 ## 搁置 / ## 归档（那两区只收散在活跃池的散户 TODO） |
| **Epic 完结** | 全部成员 🟢/⚫ 后，Epic 块**留在 ## 专项**当台账（这正是为了回顾）；想让活跃 Epic 靠前，可把完结的块挪到本区底部 |
| **推进** | "推进 Epic：<名>" 走 [todo-progress](../skill/todo-progress/SKILL.md) §推进整个 Epic：按依赖序逐个推、每条门槛照常 |

### 修改规则（什么时候改、由谁触发）

> **散户 TODO（活跃池）会迁表；Epic 成员永不迁表（原地改）**。下面两类分开看。

**散户 TODO（活跃池里的，不属于任何 Epic）：**

| 触发动作 | 改什么 | 由哪个 SOP 触发 |
|---|---|---|
| 创建新 TODO | 活跃项加一行 | [todo-create](../skill/todo-create/SKILL.md) §创建步骤 |
| `todo → in_progress` | 该行状态 emoji 改 🟡 | [todo-progress](../skill/todo-progress/SKILL.md) §状态转换 |
| `→ on_hold` | **删活跃行**，搁置表加一行（⏸️ + 原因含复活条件） | 同上 |
| `on_hold → 唤回` | **删搁置行**，移回活跃项（⚪/🟡） | 同上 |
| `in_progress → done` | **删活跃行**，归档表加一行（含完成日期 + 原因/产出） | 同上 |
| `→ abandoned` | **删活跃/搁置行**，归档表加一行，⚫ + 原因写为什么放弃 | 同上 |

**Epic 成员：**

| 触发动作 | 改什么 | 由哪个 SOP 触发 |
|---|---|---|
| 创建 Epic 成员 | 加到对应 `### Epic：` 表 | [todo-create](../skill/todo-create/SKILL.md) |
| **任何状态流转** | **原地改 emoji + 填 `原因/备注` 列，行不迁出**（done/abandoned/on_hold 都一样） | [todo-progress](../skill/todo-progress/SKILL.md) |
| 开 / 关 Epic | 加 `## 专项` 下 `### Epic：<名>` 子段；完结后留作台账（可挪本区底部） | todo-create / todo-progress |

### 不要做的事

- ❌ 改列序
- ❌ 加新列（除非走"改 schema"流程，更新本 README）
- ❌ 给活跃项分组（它就是一个扁平池；要分组就是开 Epic）
- ❌ 把 on_hold 留在活跃表（进搁置表）；把 done / abandoned 留在活跃表（进归档表）
- ❌ **把 Epic 成员迁到全局 ## 搁置 / ## 归档**（它们永远留在 Epic 表原地改状态）
- ❌ 搁置 / 归档 / Epic 表的**原因（备注）列留空**（done 可空，abandoned/on_hold 必填）
- ❌ 同一个 TODO 出现在两处（一个 TODO 只有一行）
- ❌ 不按 §排序规则 排（尤其归档凭感觉插行——必须按完成日期降序）
- ❌ 用 `2026.06.03` 这种非 ISO 格式（点号易跟版本号混）
- ❌ 在 frontmatter 加 count 字段（看现状即可）
